MultiPlaceholders

Ever wanted to use PlaceholderAPI for two or more players in the same message?
We got you!

Let's take the Welcome Actions as an example.
The are two available placeholders: %player% & %new_player%

PlaceholderAPI placeholders are written, for example, like this: 

%luckperms_prefix% %player% & %new_player%

PlaceholderAPI placeholders for the Welcome Actions are applied by default to the %player%.
If you want to add the %luckperms_prefix% also to the new player you can simply do this: 

%ph_player%%luckperms_prefix% %player% & %ph_new_player%%luckperms_prefix% %new_player%

This means that where there are more than one placeholder for the player name (%player% & %new_player% returns the name of the player), there are also placeholders for this feature. Syntax of the placeholder is: %ph_placeholder%%placeholderapi_placeholder%
