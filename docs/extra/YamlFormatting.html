YamlFormatting

For multi-line messages you can use the "\n" character but the YAML format allows you to create very long lines in a even simplier way.

Literal Style​

If you want a multi-line message you can use the "|" character:

message: |
  
  Line1
  
  Line2
  
  Line3

This example is equal to writing: “\n\nLine1\n\nLine2\n\nLine3\n“.
Every line break is present except for the ending line breaks, which are reduced to one.

Keep​

If you want to keep every ending line you can use “|“ followed by “+“:

message: |+
  Line1
  Line2
  Line3

Equal to: “Line1\nLine2\nLine3\n\n“.

Strip​

Else if you want to remove every ending line you can use “-“ instead of “+“:

message: |-
  Line1
  Line2
  Line3

Equal to: “Line1\nLine2\nLine3“.

Folded Style​

The “>“ sign let us write long single-line messages by splitting them on multiple lines:

message: >
  Line1
  Line2
  Line3

Equal to: “Line1 Line2 Line3“

Here’s another example:

message: >
  Line1
  Line2
  
  
  Line3

Equal to: “Line1 Line2\n\nLine3\n“.
As you can see empty lines are seen as line breaks and ending line breaks are reduced to one like the literal style.

The “-“ and “+“ operators can be used and they affect the folded style in the same way as the literal style.
